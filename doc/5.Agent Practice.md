5) Agent Practice (프롬프트/RAG/툴/가드레일 디버깅)

이건 요즘 “AI Engineer” 실무의 핵심이라 취업 직결됨.

어떤 문제를 주나

일부러 망가진 Agent/RAG 파이프라인을 제공

사용자는 Prompt / Retrieval / Tool schema / Memory policy 등을 수정해 개선해야 함

예시 시나리오



RAG인데 근거를 안 붙이고 환각



Tool calling이 엉뚱한 툴을 호출



Long context에서 이전 약속을 깨먹음



시스템 프롬프트가 충돌



사용자 Input



수정한 prompt



retriever top-k, chunking, query rewrite on/off



tool schema/constraints



guardrail 규칙



시스템 Output (완전 수치로)



accuracy / faithfulness(근거 일치율) / hallucination rate



latency / token cost



실패 케이스 리플레이(“이 질문에서 왜 망했는지”)



연습 목표



LLM이 틀릴 때 ‘시스템적으로’ 고치는 능력 훈련



“모든 트랙을 하나로 묶는” 가장 재밌는 방식 (너 취향) Campaign Mode: “Engineer RPG”



한 번의 캠페인에서 이렇게 이어진다:



Code: 기능 구현



Debug/Cleanup: 품질/버그 정리



System: 트래픽 늘어났다고 가정하고 아키텍처로 확장



Ops: 장애 터짐 → 복구



Agent: AI 기능 추가했더니 환각 → 안정화



이러면 “연습 플랫폼”이 아니라 게임이 됨.
