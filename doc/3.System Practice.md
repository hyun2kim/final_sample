3) System Practice (아키텍처 그리기 + 트레이드오프)

너 말대로 그림을 잘 그리는 연습이 핵심인데,
그림만 예쁘게 그리면 의미 없어. 그래서 “결과”가 있어야 함.

어떤 문제를 주나

“상황 + 요구사항 + 제약 + 트래픽”을 줌

사용자는 아키텍처(다이어그램 + 선택 이유)를 제출

예시 시나리오

“사내 문서 검색 챗봇(보안/온프렘)”

“주문/결제 이벤트 처리(peak 트래픽)”

“실시간 알림 시스템(푸시)”

사용자가 제출하는 것 (Input)

Mermaid 다이어그램(필수)

컴포넌트 설명(짧게)

트레이드오프 3개(필수)

시스템이 주는 것 (Output)

체크리스트 기반 피드백:

SPOF 있나?

병목 가능성?

캐시 전략?

장애 모드에서 어떻게 됨?

“대안 아키텍처 1개” 제안

“질문 5개” (시니어 면접관 모드)
