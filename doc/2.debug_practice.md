어떤 문제를 주나
2-A) Bug Hunt (버그 찾기)

정상처럼 보이는데 특정 케이스에서 터지는 코드

off-by-one, null guard, race condition(선택), resource leak 등

2-B) Vibe Cleanup Specialist (클린업/리팩토링)

“돌긴 도는데 지저분한 코드”

중복, 네이밍, 응집도 낮음, 예외처리 없음, 함수 너무 김

사용자가 제출하는 것 (Input)

(Bug Hunt) 수정 patch(diff)

(Cleanup) 리팩토링 코드 + “리팩토링 원칙 3줄”

시스템이 주는 것 (Output)

✅ 버그 트랙: 회귀 테스트 통과 + 수정 최소성(diff 라인 수)

✅ 클린업 트랙: 테스트 유지 + 복잡도 감소 + 중복 감소
